ModbusExtension adalah komponen non-visible untuk MIT App Inventor yang menjadikan aplikasi Android sebagai Modbus master (client). Baca dan tulis coil / register ke PLC, modul I/O, sensor industri, atau mikrokontroler — lewat Wi-Fi/Ethernet (Modbus TCP) maupun adapter USB-to-serial (Modbus RTU) — tanpa menulis satu baris kode Java/Kotlin.

Pendahuluan

Modbus adalah protokol komunikasi serial/industri yang menggunakan pola master–slave: satu perangkat master mengirim permintaan (request), dan satu atau beberapa perangkat slave membalas (response). Protokol ini banyak dipakai pada PLC, modul I/O jarak jauh, meter listrik, VFD (inverter motor), dan berbagai sensor industri karena sederhana dan sudah menjadi standar de-facto sejak tahun 1979.

ModbusExtension menempatkan ponsel/tablet Android sebagai master. Ada dua jalur komunikasi yang didukung:

  • Modbus TCP — lewat Wi-Fi atau Ethernet (via adapter), menghubungi host:port perangkat/gateway secara langsung.
  • Modbus RTU — lewat adapter USB-to-serial (CH340, CP210x, FTDI, PL2303, CDC-ACM) yang tersambung ke port OTG Android, umum dipakai untuk RS-232/RS-485.

Cocok dipakai untuk

  • • Aplikasi monitoring/HMI mobile untuk mesin produksi
  • • Dashboard SCADA ringan berbasis Android
  • • Alat bantu commissioning & troubleshooting lapangan
  • • Proyek edukasi otomasi industri & IoT

Pengenalan Komponen

Setiap blok ditampilkan berikut gambar block, tipe nilai, dan keterangan fungsinya — persis seperti tampilan di Blocks Editor App Inventor.

Saat komponen ModbusExtension1 ditambahkan ke Screen, App Inventor otomatis menyediakan blok referensi ini — dipakai untuk mengoper komponen sebagai nilai (misalnya ke prosedur kustom).

Contoh Pseudo-code Blok

Notasi berikut merepresentasikan susunan blok di Blocks Editor secara tekstual, untuk memudahkan membaca alur logika tanpa perlu screenshot per kombinasi blok.

1 · Konek TCP saat Screen dibuka

when Screen1.Initialize do
  set ModbusExtension1.SlaveId to 1
  set ModbusExtension1.ResponseTimeout to 1000
  call ModbusExtension1.ConnectTcp
    host: "192.168.1.50"
    port: 502

when ModbusExtension1.Connected do
  call ModbusExtension1.ReadHoldingRegisters
    startAddress: 0
    count: 10
    unsigned: true

2 · Menampilkan hasil pembacaan register

when ModbusExtension1.GotRegisters(values) do
  set Label_Suhu.Text to
    select list item list values index 1
  set Label_Tekanan.Text to
    select list item list values index 2

when ModbusExtension1.OperationError(operation, message) do
  set Label_Status.Text to
    join "Gagal: " operation " - " message

3 · Menulis nilai dari tombol (Write Coil)

when Button_Pompa.Click do
  call ModbusExtension1.WriteCoil
    address: 5
    value: true

when ModbusExtension1.WriteCompleted(operation, success, errorMessage) do
  if success then
    set Label_Status.Text to "Pompa ON"
  else
    set Label_Status.Text to
      join "Gagal tulis: " errorMessage

4 · Modbus RTU via USB serial

when Button_Sambung.Click do
  call ModbusExtension1.RequestUsbPermission

when ModbusExtension1.UsbPermissionGranted(deviceName) do
  call ModbusExtension1.ConnectRtu
    baudRate: 9600
    dataBits: 8
    stopBits: 1
    parity: 0

when ModbusExtension1.UsbPermissionDenied do
  set Label_Status.Text to "Izin USB ditolak"

Fitur & Kegunaan

Karena Modbus adalah standar terbuka, ekstensi ini bisa dipasangkan dengan berbagai perangkat selama perangkat tersebut mengimplementasikan Modbus TCP/RTU sebagai slave.

ESP8266 / ESP32

Firmware Modbus TCP slave (mis. via library ModbusIP/ModbusRTU) menjembatani sensor ke Wi-Fi. Aplikasi App Inventor tinggal ConnectTcp ke IP modul.

AVR (Arduino Uno/Mega)

Berjalan sebagai Modbus RTU slave lewat RS-232/RS-485 (library ModbusSerial). Android tersambung via adapter USB-to-serial + kabel OTG.

STM32

Cocok untuk modul I/O industri custom dengan stack Modbus TCP/RTU sendiri (mis. FreeMODBUS), dibaca langsung sebagai register/coil dari HP.

PLC (Delta, Schneider, LOGO!, dst.)

Banyak PLC menyediakan Modbus TCP/RTU bawaan atau lewat modul komunikasi tambahan — dibaca sebagai HMI mobile ringan tanpa software SCADA berat.

Kegunaan umum

• Dashboard monitoring suhu/tekanan/level realtime

• Kontrol jarak dekat pompa, motor, katup (coil)

• Data logger portabel untuk audit/commissioning

• Alat bantu training otomasi & tugas akhir mahasiswa

Catatan & Batasan

  • Alamat register/coil di perangkat lapangan kadang dimulai dari 0 dan kadang dari 1 (tergantung dokumentasi vendor) — sesuaikan startAddress saat hasil pembacaan tidak sesuai.
  • Untuk Modbus RTU, pastikan baudRate, dataBits, stopBits, dan parity sama persis dengan konfigurasi slave, dan hanya satu adapter USB serial yang didukung dalam satu waktu.
  • Naikkan Pause dan ResponseTimeout apabila memakai gateway RS-485 yang lambat atau banyak slave dalam satu bus.
  • Modbus TCP standar tidak memiliki enkripsi/otentikasi — hindari mengeksposnya langsung ke internet publik tanpa VPN/firewall.
  • Selalu tangani event OperationError dan WriteCompleted agar aplikasi tidak "diam" saat koneksi putus atau slave tidak merespons.
  • Jika KeepConnection diaktifkan, ingat memanggil Disconnect() saat Screen ditutup untuk melepas socket/port secara bersih.
